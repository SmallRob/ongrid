# req-system-mapping 操作指令 (简化版)

## 目的

检查特性列表中的外部服务依赖,生成集成检查清单和风险提示。适用于 Nice Today 2.0 项目中涉及 Gemini AI、Capacitor 插件、Three.js 等外部服务的场景。

---

## 语言检测

在生成任何内容之前,必须检测并使用当前环境的响应语言(中文/英文)。所有输出必须保持语言一致性。

---

## 适用场景

| 场景 | 说明 |
|------|------|
| 需求涉及 Gemini AI 集成 | 数字分身对话、智能分析、内容生成 |
| 需要使用 Capacitor 原生能力 | 相机、定位、本地存储、推送通知 |
| 包含 Three.js 3D 渲染 | 虚拟展示、能量树、粒子效果 |
| 集成第三方 SDK | 支付、分享、统计、地图 |

---

## 上下文注入

### 必需读取的文件

1. **特性列表** (必需)
   - 路径: `.asdm/workspace/requirements/REQ-<id>-<name>/feature-list.md`
   - 用途: 提取特性的 `external_dependencies` 字段

2. **澄清记录** (推荐)
   - 路径: `.asdm/workspace/requirements/REQ-<id>-<name>/clarification-notes.md`
   - 用途: 了解业务背景和集成需求

---

## 系统映射步骤

### 步骤 1: 接收需求 ID

从用户处获取需求 ID (例如: `REQ-001-数字分身对话`)

### 步骤 2: 校验特性列表

检查 `feature-list.md` 是否存在,如不存在提示用户先运行 `/req-breakdown`

### 步骤 3: 提取外部服务依赖

从特性列表中提取所有 `external_dependencies` 字段,按类型聚合:

```
外部服务依赖汇总:
- AI 服务: Gemini API (F-001, F-003)
- 设备 API: Capacitor Camera (F-002), Capacitor Storage (F-001)
- 3D 渲染: Three.js (F-004)
```

### 步骤 4: 生成集成检查清单

针对每种依赖类型,生成检查清单:

#### AI 服务集成检查 (Gemini API)

- [ ] API Key 配置和安全管理
- [ ] Token 使用量监控和成本控制
- [ ] 响应格式定义和错误处理
- [ ] 网络超时和重试机制
- [ ] 离线降级方案(如:缓存历史回复)

#### 设备 API 集成检查 (Capacitor)

- [ ] 权限申请流程设计(相机/定位/存储)
- [ ] Android 平台兼容性测试
- [ ] Web 平台降级方案(如:Web API 替代)
- [ ] 用户拒绝权限后的功能降级

#### 3D 渲染集成检查 (Three.js)

- [ ] FPS 监控和性能优化(目标 60fps)
- [ ] 内存管理和资源释放
- [ ] 低端设备降级方案(如:静态图片替代)
- [ ] 加载状态和用户体验优化

### 步骤 5: 输出集成检查文档

保存至: `.asdm/workspace/requirements/REQ-<id>-<name>/system-function-list.md`

**文档结构**:
```markdown
# 外部服务集成检查清单

## 需求编号: REQ-<id>-<name>

## 外部服务依赖汇总
(按类型聚合的依赖清单)

## 集成检查清单
(按服务类型的详细检查项)

## 风险评估
(高/中/低风险及缓解措施)

## 权限申请清单
(Capacitor 插件所需权限)

## 下一步建议
- 优先处理高风险依赖
- 制定降级方案
- 进行技术可行性验证
```

## 执行指南

### 何时使用本操作

- 特性列表中包含外部服务依赖时
- 需要评估 AI/Capacitor/Three.js 集成风险时
- 需要制定权限申请和降级方案时

### 输出示例

```
# 外部服务集成检查完成

## 需求编号: REQ-001-数字分身对话

### 外部服务依赖汇总
| 服务类型 | 服务名称 | 涉及特性 | 风险等级 |
|---------|---------|---------|---------|
| AI 服务 | Gemini API | F-001, F-003 | 高 |
| 设备 API | Capacitor Storage | F-001 | 中 |

### 关键风险
1. **Gemini API 调用成本** - 建议实施 Token 用量监控
2. **网络不稳定** - 建议实施离线缓存和重试机制

### 下一步
- 完成技术可行性验证
- 制定详细的降级方案
- 更新需求文档的"外部服务集成说明"章节
```
