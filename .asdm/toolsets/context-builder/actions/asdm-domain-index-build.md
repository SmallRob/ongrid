# asdm-domain-index-build: 构建领域索引

## 目的

为指定领域生成 L2 领域索引（< 1KB），建立领域级的轻量级导航系统。

**前置条件**：已执行 `/asdm-context-init` 生成了 L1 顶层索引

## 语言检测

在生成任何内容之前，必须检测并使用当前环境的响应语言。

## 上下文粒度原则

**L2 领域索引设计原则**：
- 大小限制：< 1KB/领域
- 一句话描述领域职责
- 子模块列表 + 关键文件入口
- 外部依赖关系

## 输入参数

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 领域名 | String | 是 | 要构建索引的领域名称 |

## 构建步骤

### 步骤 1: 定位领域

1. 读取 `.asdm/contexts/index.md` 了解项目全局结构
2. 确认指定领域是否在项目 L1 索引中定义
3. 如果领域不存在，提示用户先分析项目结构

### 步骤 2: 分析领域代码

分析指定领域的代码结构：

1. **确定领域边界**
   - 领域包含哪些子模块/包
   - 领域的核心职责是什么
   - 领域与其他领域的边界

2. **识别子模块**
   - 子模块名称和职责
   - 关键类和文件
   - 模块间关系

3. **识别外部依赖**
   - 依赖哪些其他领域
   - 依赖哪些基础设施
   - 提供哪些接口给其他领域

### 步骤 3: 生成 L2 领域索引

使用 `specs/layer-2/domain-index.md` 模板生成领域索引：

**核心要求**：
- **大小限制**：< 1KB
- **一句话描述**：这个领域做什么
- **边界说明**：负责什么、不负责什么
- **子模块表**：3-7 个子模块，简洁描述
- **外部依赖**：依赖关系和入口链接

**输出文件**：`.asdm/contexts/domains/<领域名>-domain.md`

### 步骤 4: 创建领域详细内容目录

为该领域创建 L3 详细内容目录结构：
```
.asdm/contexts/domain-details/<领域名>-domain/
├── index.md          # 详细内容索引（由 asdm-domain-context-build 生成）
├── entities.md       # 实体定义
├── services.md       # 服务接口
├── apis.md           # API 定义
└── flows.md          # 业务流程
```

### 步骤 5: 更新 L1 索引

在 `.asdm/contexts/index.md` 的领域列表中，添加/更新该领域的入口链接。

## 输出摘要

| 文件 | 大小 | 说明 |
|------|------|------|
| `.asdm/contexts/domains/<领域名>-domain.md` | < 1KB | L2 领域索引 |
| `.asdm/contexts/domain-details/<领域名>-domain/` | - | L3 详细内容目录 |

## 使用方法

1. 用户执行 `/asdm-domain-index-build <领域名>`
2. AI 分析领域代码结构
3. AI 生成 L2 领域索引
4. 用户通过 `/asdm-domain-context-build` 按需生成 L3

## 批量构建

如需批量构建多个领域索引，可以在一次调用中指定多个领域：
```
/asdm-domain-index-build <领域1>,<领域2>,<领域3>
```

## 注意事项

1. **保持精简**：L2 索引必须 < 1KB
2. **聚焦边界**：清晰定义领域边界，避免模糊地带
3. **导航导向**：提供到 L3 详细内容的入口
4. **依赖明确**：清晰标注与其他领域的依赖关系
