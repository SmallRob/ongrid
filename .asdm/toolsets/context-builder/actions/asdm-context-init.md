# asdm-context-init: 初始化项目顶层索引

## 目的

初始化项目顶层索引（L1），建立轻量级的项目全局认知系统。
**本 action 只生成 L1 顶层索引（< 2KB），不生成详细内容。**

## 语言检测

在生成任何内容之前，必须检测并使用当前环境的响应语言：

1. **检测响应语言**：分析环境设置确定主要语言
2. **应用语言一致性**：所有生成内容使用检测到的语言
3. **支持的语言**：English (en)、Chinese (zh)

## 上下文粒度原则

**重要**：遵循三明治架构原则
- L1 顶层索引：< 2KB，极精简
- L2 领域索引：< 1KB/领域，按需生成
- L3 详细内容：< 5KB/文件，执行时按需生成

## 初始化步骤

### 步骤 0: 发现领域（可选但推荐）

**重要**：在初始化之前，建议先使用 `/asdm-domain-discovery` 发现领域。如果没有执行此步骤，AI 会在初始化过程中自动分析。

#### 如何发现领域？

**核心原则**：基于项目**实际的代码结构**识别领域，不要套用预设模板。

AI 会通过以下方式识别领域：

1. **扫描实际代码结构**
   - 扫描 `src/` 目录下的实际模块/包结构
   - 识别 Maven/Gradle 子模块
   - 列出所有实际的目录和包名

   ```bash
   # 示例：扫描得到的结果可能完全不同
   # 微服务项目：user-service, loan-service, insurance-service
   # 电商项目：product-center, inventory-center, promotion-center
   # ERP项目：hr-module, finance-module, workflow-module
   ```

2. **分析每个模块的实际职责**
   - 分析核心类/接口定义
   - 读取 README 或模块说明
   - 通过代码注释和 docstring 理解职责

   ```
   ┌─────────────────────────────────────────────────────────┐
   │  示例：基于实际结构识别                                 │
   │                                                         │
   │  扫描得到: loan-service/                                │
   │  分析发现: 包含 LoanApplication, LoanApproval 等类      │
   │  结论: 这是一个"贷款服务"模块                           │
   │                                                         │
   │  不要预设: "一定有用户域/订单域"                        │
   └─────────────────────────────────────────────────────────┘
   ```

3. **统计代码量分布**
   - 统计各模块/包的代码行数
   - 评估领域的大小和复杂度

4. **生成基于实际结构的领域列表**
   - 输出**项目实际存在**的模块及其职责
   - 供用户确认或调整

#### 错误做法 vs 正确做法

```
❌ 错误做法：套用预设模板
"项目一定有用户域、订单域、支付域..."

✓ 正确做法：基于实际结构
"扫描发现项目有 loan-service, insurance-service...
 这些是基于实际代码识别的领域"
```

### 步骤 1: 创建目录结构

创建以下目录结构：
```
.asdm/contexts/
├── domains/           # L2: 领域索引
│   └── placeholder.md  # 占位，说明需要使用 /asdm-domain-index-build 生成
├── domain-details/     # L3: 详细内容
│   └── placeholder.md  # 占位，说明需要按需生成
└── workspace/         # 工作区汇总
```

### 步骤 2: 分析项目结构

分析工作区以了解：

1. **项目基本信息**
   - 项目名称和描述
   - 项目规模（代码行数、模块数量）

2. **技术栈识别**
   - 主要编程语言
   - 核心框架和库
   - 数据库和中间件
   - 构建工具

3. **领域划分识别**
   - 识别业务领域（如：用户、订单、支付）
   - 识别基础设施领域（如：缓存、消息、搜索）
   - 识别跨域领域（如：认证、日志）

4. **构建命令识别**
   - 构建命令
   - 测试命令
   - 启动命令

### 步骤 3: 生成 L1 顶层索引

使用 `specs/layer-1/project-index.md` 模板生成 `index.md`：

**核心要求**：
- **大小限制**：< 2KB
- **一句话描述**：用一句话说明项目是什么
- **技术栈表**：简洁，核心 5-7 项
- **领域列表**：5-10 个领域，包含入口链接
- **构建指南**：3-5 个核心命令

**输出文件**：`.asdm/contexts/index.md`

### 步骤 4: 生成占位文件

在 `domains/` 和 `domain-details/` 目录生成占位文件，说明如何按需生成上下文：

```markdown
# 领域索引

本目录包含各领域的 L2 索引文件。

使用以下命令生成领域索引：
/asdm-domain-index-build <领域名>

当前已生成的领域：
- [列出已生成的领域]

未生成的领域：
- [列出未生成的领域]
```

## 输出摘要

完成初始化后，生成以下产物：

| 文件 | 大小 | 说明 |
|------|------|------|
| `.asdm/contexts/index.md` | < 2KB | L1 项目顶层索引 |
| `.asdm/contexts/domains/` | - | L2 领域索引目录 |
| `.asdm/contexts/domain-details/` | - | L3 详细内容目录 |

## 使用方法

1. 用户执行 `/asdm-context-init`
2. AI 分析项目结构
3. AI 生成 L1 顶层索引
4. 用户通过 `/asdm-domain-index-build` 按需生成 L2/L3

## 注意事项

1. **保持精简**：L1 索引必须 < 2KB，不要试图包含过多细节
2. **留有扩展**：L1 索引是导航系统，详细内容在 L2/L3
3. **领域划分**：按"业务决策域"划分，而非按代码结构
4. **增量构建**：后续使用 `/asdm-domain-index-build` 按需生成
